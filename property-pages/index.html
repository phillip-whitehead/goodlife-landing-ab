How It Works

The Weekly Cycle

Every Monday morning, one command pulls all the data, generates the briefing, and commits it to the repo. No manual data entry.

1️⃣
Pull Data
GA4 pulls sessions, bounce rate, avg duration, and form-submit events for every /property/* path. HubSpot pulls pipeline deals and conversion rates.
2️⃣
Rank Properties
Properties are ranked by inquiry rate (form submits ÷ sessions). Top 5 best and worst performers are identified. Minimum 10 sessions required to qualify.
3️⃣
Derive Hypothesis
A decision tree checks inquiry rate, mobile bounce, avg session time, winner vs. loser gap, and close-lost rate to pick the most pressing lever and generate a testable hypothesis.
4️⃣
Generate Briefing
A styled HTML briefing document is created with summary stats, property rankings, the hypothesis, and 5 concrete action items for the week. Saved as week-N.html.
5️⃣
Log & Deploy
Metrics are appended to GoodLife_Property_CRO_Tracker.xlsx, this index is updated with a new week card, and everything is committed and pushed to Cloudflare Pages automatically.
6️⃣
Manual Review
Review the briefing, check Clarity recordings for the worst property, implement the recommended changes on the live site, and track the impact the following Monday.
